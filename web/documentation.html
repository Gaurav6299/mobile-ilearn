1–Overview

MiLearn is a streamlined, mobile version of Blackboard iLearn. It is currently available for Android devices.

1.1–Requirements

Have MiLearn app downloaded.

1.2–Development Process

During development of MiLearn, we used a Scrum process of development. Our group would meet up at least once a week on Mondays for discussing our contributions towards the project, as well as planning on what part of the project to do next.

In addition, we took a modular approach towards development. Each group member focused on a different component of the app. This boosted overall group efficiency as we were able to concentrate on one item to do per week instead of several. One problem was actually getting each component integrated properly each week.

1.3–Development Tools Used

In order to build MiLearn, the following languages, tools, and APIs were used.

  • Languages
    Java
    Used for actual app developement.
    XML
    Used for app layout.

    Lets JavaScript functions call Java methods.

    Framework for mobile web app.

2–User Manual

This is a quick reference for using the MiLearn app.

For those who wish to know how the MiLearn app, please look here first.

2.1–Logging In/Logging Out

When MiLearn is first opened, the Login screen is brought up. Here, the user will need to insert their username (i.e. UCR NetID) and password.

2.2–Home Screen

Once the user has logged in, they are brought to the Home Screen. This will initially display the "Welcome to MiLearn" screen.

The Home Screen should then display the user's name and any alerts that they may have.

Alerts are coded such that high priority alerts are in red, medium priority in yellow, and low priority in green.

The timing of each priority can be set in the Settings Screen.

2.3–Classes Screen

The Classes Screen displays an accordion for all of the user's classes for the current quarter. The header of each collapsible contains the course number followed by the course section. Inside of each collapsible is the full name of the course.

If the user is a student, then they will see what class(es) that they are enrolled in. In addition the name(s) of the instructor(s) are displayed inside the collapsible.

If the user is an instructor, then they will see what class(es) they are teaching. The collapsible content also includes a list of students that are enrolled in that course.

2.4–Assignments Screen

The Assignments Screen displays an accordion with the assignments ordered by due date (from sooner to later). Each collapsible header has the course that the assignment is for followed by the assignment title.

Students can view what assignments are due for all classes that they are enrolled in. They can also upload a file to turn in for an assignment. Assignment uploads can be done multiple times, but the instructor will only see the most recent submission. Currently, students can only upload one file at a time.

Instructors can create new assignments or edit assignments for any class they teach. This is respectively done through an already existing assignment collapsible or through the "New Assignment" collapsible at the bottom of the list.

2.5–Grades Screen

The Grades Screen displays an accordion with the grades for each assignment in each class.

For students, each collapsible is for a course that they are enrolled in. The collapsible heading indicates which course the student they will be looking in while the content displays (in chronological order), the grades for each assignment.

Instructors each collapsible is for an assignment that they have assigned and is ordered chronologically. Inside each collapsible is the name of the assignment, how many points it was worth, and a table of students' grades for the assignment. The total points and each student grade can be edited.

2.6–Settings Screen

The Settings Screen allows all users to configure the theme of their user interface and the time for priority levels.

A quick guide for themes:

Theme Description
Default Generic gray/chrome theme.
Dark Generic dark theme.
Highlander Show UCR school spirit.
Go Blind! For trolls and schmucks only.
Super! Basic! ... why?

Priority levels should reflect the user's feelings about how soon they should begin an assignment or how late they should be when finishing one.
